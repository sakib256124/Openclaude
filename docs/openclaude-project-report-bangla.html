Project Report

Openclaude / OpenCloud Compute Console

Ubuntu Multipass virtual machine manage করার জন্য EC2-inspired secure web dashboard.

Project type: University final/project demonstration

Application name: OpenCloud Compute Console

Workspace/folder name: Openclaude

Core stack: Next.js, TypeScript, PostgreSQL, Prisma, Auth.js, Tailwind CSS

1. Project Overview

Openclaude project-এর main application হলো OpenCloud Compute Console. এটি একটি cloud compute management dashboard, যেখানে user browser থেকে virtual machine এবং related cloud resource manage করতে পারে। Project-টি AWS EC2-এর concept দ্বারা inspired, কিন্তু এটি AWS-এর official code, API, logo বা console copy করে না। Backend target করা হয়েছে Ubuntu Multipass VM management layer।

Short Definition

OpenCloud Compute Console হলো Ubuntu Multipass VM launch, list, start, stop, restart, delete, monitor এবং database-backed cloud resource organize করার জন্য একটি secure web console prototype।

2. Problem Statement

সাধারণভাবে VM manage করতে terminal command জানা লাগে। যেমন Multipass ব্যবহার করলে user-কে multipass launch, multipass list, multipass stop ইত্যাদি command চালাতে হয়। Non-expert user বা academic lab environment-এ এটা কঠিন এবং ভুল command দিলে resource loss হতে পারে।

Existing Problem

  • VM command line দিয়ে manage করতে হয়।
  • User, role, audit log এবং permission centralized থাকে না।
  • Resource status এক জায়গায় দেখা যায় না।
  • Destructive operation যেমন delete-এর আগে UI confirmation থাকে না।

Project Solution

  • Browser-based dashboard তৈরি করা হয়েছে।
  • Authentication ও role-based permission আছে।
  • Instance, image, network, volume, billing, logs এক console-এ আছে।
  • Safe API response ও confirmation workflow রাখা হয়েছে।

3. Objectives

Main Objective

Ubuntu Multipass VM এবং related cloud resources manage করার জন্য secure, role-based, web-based cloud compute console তৈরি করা।

Specific Objectives

  1. User login, logout, registration এবং password reset workflow তৈরি করা।
  2. Role অনুযায়ী resource read/write/manage permission enforce করা।
  3. VM instance list, launch, details, start, stop, restart, terminate workflow তৈরি করা।
  4. Images, key pairs, networks, security groups, volumes, snapshots, floating IP-এর page/API তৈরি করা।
  5. Dashboard, monitoring, billing estimate এবং activity logs দেখানো।
  6. Multipass unavailable হলে app crash না করে safe local/unavailable response দেখানো।

5. System Architecture

এই project-এ browser কখনো সরাসরি Multipass command চালায় না। Browser শুধু internal API route call করে। API route user authentication, permission এবং validation check করে server-only Multipass module call করে।

1. User browser থেকে action দেয়
2. Next.js page অথবা React component local API call করে
3. API route session এবং role permission verify করে
4. Server-only Multipass service CLI command execute করে
5. Runtime response normalized DTO আকারে frontend-এ ফেরত যায়
Browser -> Next.js frontend -> /api/* route handler -> requireApiPermission() -> server-only Multipass module -> multipass CLI / local fallback -> normalized JSON response

Database full Multipass runtime duplicate করে না; বরং user, ownership, permission, metadata, billing estimate এবং audit information রাখে। VM-এর live state Multipass থেকে sync করা হয়।

8. Authentication & Permission

Project-এ Auth.js credentials login ব্যবহার করা হয়েছে। Password bcrypt hashing দিয়ে verify করা হয়। User active না হলে login/session reject হয়। API route-এ centralized requireApiPermission helper ব্যবহার করা হয়েছে।

Roles

  • ADMIN: users, settings, pricing, all logs and resources manage করতে পারে।
  • USER/DEVELOPER: own resources read/write করতে পারে।
  • VIEWER: resource read করতে পারে, destructive write করতে পারে না।

Security Controls

  • Server-only Multipass modules
  • Zod request validation
  • Encrypted optional host secret storage
  • Safe audit log messages
  • Confirmation dialog for destructive actions

9. API & Multipass Integration

Instance API route বাস্তবে Multipass CLI call করার চেষ্টা করে। যদি host machine-এ Multipass installed না থাকে, তাহলে structured safe response বা local fallback ব্যবহার করা হয়, যাতে demo/app crash না করে।

Example: Launch Instance Flow

  1. User launch form submit করে।
  2. /api/instances POST route request পায়।
  3. Zod দিয়ে name, image, CPU, memory, disk validate হয়।
  4. Permission resources:write check করা হয়।
  5. Multipass command multipass launch execute হয়।
  6. Database-এ ComputeInstance record upsert হয়।
  7. UI instance response দেখায়।
multipass launch 24.04 --name demo-vm --cpus 2 --memory 2G --disk 10G multipass list --format json multipass info demo-vm --format json multipass stop demo-vm multipass delete demo-vm --purge

12. Limitations

Current Limitation

  • Multipass host machine-এ installed না থাকলে real VM create হবে না; local fallback/unavailable response দেখাবে।
  • Security group rules database/API level-এ আছে; real host firewall automation আলাদা কাজ।
  • Multipass AWS EBS-style block volume directly দেয় না; mount-based implementation দরকার।
  • True historical metrics-এর জন্য periodic collector বা guest agent লাগবে।
  • Browser SSH terminal-এর জন্য websocket/session broker দরকার।
